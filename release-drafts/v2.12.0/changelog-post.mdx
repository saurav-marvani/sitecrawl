## v2.12.0 is live

Firecrawl v2.12.0 lands a new set of enterprise controls — URL-level threat protection backed by Google Web Risk with per-URL billing, per-team API key IP allowlists, per-key format/endpoint lockdown, and per-key usage limits enforced through Autumn — plus session replay for `/v2/interact` with per-tab HLS playlists, per-key concurrency limits on the FoundationDB queue, and a `search` monitor target with a goal-driven LLM judge stack, flat deterministic billing, and stable event dedupe. It also adds a first-party Slack App with a `/monitor` slash command, PHP SDK tool classes for Laravel's AI framework, a `menu` scrape format gated behind `menuBeta`, a hosted parse-upload flow for MCP tools, PDF `totalPages` for truncation detection, an `enterprise` search parameter across the JS/Python/Ruby/.NET SDKs, batched `/v2/search` highlights, a `SELF_HOSTED_DOMAIN` env var behind reverse proxies, transparent gzip decompression on Fire Engine scrapes, an Express 5.2 + TypeScript 7 stable build, and finishes the migration to Autumn as the single source of truth for credit accounting across v0, v1, and v2.

### Highlights

- **URL-level threat protection** — Enterprise, flag-gated (`threatProtection: "disabled" | "allowed" | "forced"`) domain- and URL-risk classifier that enforces across scrape, batch scrape, crawl, search, map, extract, and agent on v1 and v2. `normal` mode classifies against Google Web Risk's Update API with locally synced hash-prefix lists and per-URL host-suffix × path-prefix expressions, so a flagged page is caught even on an otherwise clean domain. Redirects are re-checked on any URL change, blocked responses raise `unsafe_domain_blocked`, verdicts are never persisted (ZDR), and scans bill at +2 credits per unique URL.
- **IP allowlists on API keys** — Enterprise `ipRestriction` team flag rejects API-key and OAuth (`fco_`) requests from any IPv4/IPv6/CIDR entry not on the team's `ip_restriction_config` list, with a 60 s Redis-cached lookup, fail-closed semantics, and a dashboard admin cache-clear endpoint. Empty/missing lists remain unrestricted so a team can't lock itself out before configuring.
- **Per-key scope and format lockdown** — Enterprise `keyRestriction` flag lets teams lock individual API keys to allowlists of output formats and endpoint groups — a locked key handed to an agent environment cannot be promptable into fetching raw HTML, using `screenshot` / `scrape` / `executeJavascript` / `pdf` actions, or calling other endpoints. Unknown paths deny by default, v0 is denied outright, and enforcement runs server-side with no per-request override.
- **Per-key usage limits via Autumn** — API-key spend limits (the "Set limit" flow in the dashboard) are now enforced end-to-end: every `checkCredits` call site forwards `apiKeyId` to Autumn, so a request from a key that has exceeded its configured daily/weekly/monthly limit is denied with 402 even when the team as a whole still has credits.
- **Session Replay for `/v2/interact`** — New `GET /v2/interact/:sessionId/replay` and `GET /v2/interact/:sessionId/replay/:pageId` return per-tab metadata and HLS `.m3u8` playlists (with pre-signed ~6 h segment URLs). Recording is on by default; pass `"recordSession": false` on create to opt out, and `scrape-interact` responses now include `sessionId` so callers can link a job to its replay.
- **`search` monitor target** — Added a first-class `search` monitor target accepting `queries[]`, `searchWindow`, `includeDomains[]`, `excludeDomains[]`, and `maxResults`, with `raw` (no LLM) and `deep` (LLM judge) depths derived from `judgeEnabled`, a `material_dev` alert mode that re-alerts on material developments of a known event, and stable opaque event keys to prevent label-drift duplicates. Exposed across the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs with a `MonitorSearchTargetResult` type.
- **Search monitor billing** — Search monitors now bill at a flat per-check rate: `SEARCH_CREDITS` is charged `ceil(totalResults / 10) * 2`, judged results are charged 1 credit per result (down from 5), and `raw` mode pays nothing for the judge. Deep-mode per-page scrapes flow through the shared scrape pipeline as queued jobs billed under `CREDITS`, not double-counted on the search check.
- **Slack App** — Connect a Slack workspace and receive monitor alerts as Block Kit summaries, or drive monitors from Slack with the `/monitor` slash command (`watch`, `list`, `cancel`, `status`, `help`). OAuth, signed webhooks, encrypted bot-token storage, auto-join of public channels, and a `notification.slack` block on monitors are all included.
- **PHP SDK Laravel AI tools** — Native `FirecrawlScrape`, `FirecrawlSearch`, `FirecrawlMap`, and `FirecrawlCrawl` tool classes drop into any [laravel/ai](https://laravel.com/docs/13.x/ai-sdk) agent's `tools()` array — same tool names as the MCP server, output size budgets that keep results model-sized, and container-resolved auth reuse. Ships as `firecrawl/firecrawl-php@1.9.0` with the release pipeline now cutting proper Packagist tags.
- **`menu` scrape format** — A new `formats: ["menu"]` returns deterministic structured restaurant-menu data as `document.menu`, mirroring the existing `product` format end-to-end. Beta-gated via the `menuBeta` team flag with a `MENU_EXTRACTION_SERVICE_URL` self-host kill-switch, available across the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs.
- **Hosted `/v2/parse` upload flow** — `POST /v2/parse/upload-url` mints a short-lived, team-scoped `uploadRef` plus signed storage upload parameters; the client uploads bytes directly to GCS; then `POST /v2/parse` accepts JSON with `uploadRef` and parses normally. Refs are HMAC-signed with a 10-minute expiry and bound to team, object path, filename, content type, and max size, so MCP clients can parse local files without exposing an API key to the model or shell.
- **Per-key concurrency limits on the FDB queue** — Added a third gate to the FoundationDB queue's concurrency chain (crawl → key → team → ready) driven by a new nullable `api_keys.concurrency` column. Keys with a limit set hold a per-key slot while queued/active; crawl child jobs inherit the kickoff key so whole crawls stay gated; keys without a limit are fully unaffected.
- **PDF `totalPages` metadata** — PDF scrapes now return `metadata.totalPages` alongside `numPages`, so callers can tell a truncated result (`totalPages > numPages`) from a full parse. Threaded through the API, index cache, and JS/Python SDK types.
- **`enterprise` search parameter across SDKs** — The `/v2/search` `enterprise` option (e.g. `["zdr"]`, `["anon"]`) now works from JS, Python, Ruby, and .NET (`@mendable/firecrawl-js@4.29.0`, `firecrawl@4.31.0` Python, `firecrawl@1.10.0` Ruby, `firecrawl-sdk@1.9.0` .NET). Previously JS/Python silently dropped the field and .NET typed it as `bool?`.
- **`threatProtection` in JS & Python SDKs** — The v1/v2 `threatProtection` per-request override is now exposed in `@mendable/firecrawl-js@4.30.0` and `firecrawl-py@4.32.0` on scrape, batch scrape, crawl (via `scrapeOptions`), search, map, extract, and agent — no more raw HTTP for enterprise callers.
- **JS SDK `webhook` on extract** — `startExtract` and `extract` in the JS SDK now accept `webhook?: string | WebhookConfig | null`, matching `crawl` and `batchScrape`.
- **Batched `/v2/search` highlights** — The `highlightsBeta` snippet path now sends every indexed web and news result to the model service in a single `/batch_highlight` request via a shared indexed-Chain helper, mapping responses back by stable page ID with a provider-snippet fallback on missing/empty outputs. A separate `HIGHLIGHT_SHADOW_RATE` / `HIGHLIGHT_SHADOW_MAX_INFLIGHT` shadow path can sample real traffic without mutating responses.
- **`SELF_HOSTED_DOMAIN` env var** — Self-hosted instances behind a reverse proxy can now set `SELF_HOSTED_DOMAIN` (or send `X-Forwarded-Host`) so crawl status URLs, batch-scrape URLs, parse-upload links, and pagination `next` links use the externally-reachable host instead of the internal bind address.
- **Transparent gzip decompression** — Fire Engine scrapes now detect gzipped payloads by magic bytes and transparently decompress them (up to 50 MB), so callers no longer have to pre-inflate `.gz` documents.
- **Express 5 and TypeScript 7 stable** — The API and worker now run on Express 5.2.1 with `express-ws` reapplied to the v1/v2 routers, and the API build compiler is `typescript@7.0.2` stable for roughly 3× faster clean builds (median ~1.3 s vs ~5.60 s on TS 5.9.2). A side-by-side `typescript → @typescript/typescript6@6.0.2` alias keeps knip and other classic-API consumers working.
- **Autumn is the single source of truth for credits** — Removed the last ACUC-based credit accounting paths from the v0/v1/v2 hot path (including the `fireclaw` multi-play check and the unverified agent-key 50-credit cap), and pointed the API's ACUC RPCs at `_55` — giving single-team orgs `GREATEST(override, plan)` concurrency, pay-as-you-go orgs a concurrency floor of 5, per-endpoint `rate_limits` merges with the base plan, multi-team-org fallback to Scale/Enterprise limits, and stripping the last six zeroed credit columns plus five deprecated subscription/price fields from the auth chunk.
- **Lazy-loaded images** — Fixed the largest-image picker to fall back to `data-srcset` and `data-src` when `srcset` is missing, so below-the-fold images on lazy-loading sites come back with the real URL instead of a 5px placeholder.
- **`proxy: enhanced` / `stealth`** — Fixed `proxy: enhanced` and `proxy: stealth` scrapes silently routing through non-stealth engines on the cloud — the fallback list is now restricted to stealth-capable engines when stealth is requested.
- **Playwright cookie redirect fix** — Playwright-engine scrapes now seed the `Cookie` header into the browser context via `context.addCookies()` scoped to the registrable domain, so authenticated pages that 302 (including cross-subdomain hops) no longer come back as the login page.
- **CVE patches** — Resolved multiple CVEs by upgrading `multer` to `2.2.0` (GHSA-3p4h-7m6x-2hcm, GHSA-72gw-mp4g-v24j), `undici` to `7.28.0` with a pnpm override locking it major-bound (GHSA-35p6-xmwp-9g52, GHSA-g8m3-5g58-fq7m, GHSA-hm92-r4w5-c3mj, GHSA-p88m-4jfj-68fv, GHSA-pr7r-676h-xcf6, GHSA-vmh5-mc38-953g, GHSA-vxpw-j846-p89q), `systeminformation` to `5.31.17` (GHSA-5xpp-75jx-m839 — OS command injection in `networkInterfaces()`), and `js-yaml` to `3.15.0` via a major-bound pnpm override (GHSA-h67p-54hq-rp68), plus removing the malicious `serpapi-python==0.1.5` pin from the GPT-4.1 company researcher example in favor of `google-search-results`.
